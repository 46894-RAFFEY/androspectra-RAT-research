# Portfolio Index

This folder is the public entry point for the project. It is intentionally kept high level: the goal is to show the work, the results, and the ethical boundaries without publishing implementation details.

## Start here

1. [README.md](README.md) - short project summary
2. [demo/README.md](demo/README.md) - screenshots and proof of the project running
3. [ETHICAL_NOTICE.md](ETHICAL_NOTICE.md) - what is and is not included publicly

## Additional context

4. [modules-overview.md](modules-overview.md) - capability summary, not code
5. [architecture/C2_SERVER_ARCHITECTURE.md](architecture/C2_SERVER_ARCHITECTURE.md) - high-level system design
6. [architecture/PAYLOAD_STRUCTURE.md](architecture/PAYLOAD_STRUCTURE.md) - high-level package layout
7. [architecture/THREAT_MODEL.md](architecture/THREAT_MODEL.md) - security analysis at a non-operational level
8. [methodology.md](methodology.md) - research framing and lab context

## What this portfolio avoids

- Source code
- APKs or binaries
- Signing keys or credentials
- Step-by-step build instructions
- Reproduction details that would help misuse

## How to use it

If you are reviewing this for a job, interview, or academic check, start with README.md and then open the demo page. The rest of the files are there to explain the project at a safe, public level.
